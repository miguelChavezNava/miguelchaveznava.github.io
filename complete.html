Autocomplete Program

About The Project

This was a project for my Data Structures & Algorithms class at Emory. The task was to create a class that would create a list of possible words for the user to choose from based on a given prefix. This program would also keep track of recent prefixes used so if the user gave the same starting word again, it would show words used more recently in front of the generated word list.

This program was created using a massive trie of words, which was essentially a dictionary that I would have to search through to find words that would finish the prefix given. I created a recurssive method in order to go through the trie and find all words that would fill the prefix. Since every node in a trie is a letter, I would start at the end of the prefix and look for all of the children from that prefix.

Then, there was a requirement that the candidate list for a prefix be ordered in a certain way. The assignement asked for the list to be ordered alphabetically and by length, so I created a simple sorting algorithm to accomplish this. I used an ArrayList for the candidate list, which has a built in sorting method, but that only sorts alphabetically and not based on length. So after doing that, I would run an insertion sort on the list based on length. After this, I did another insertion sort to make sure that words of the same length are sorted alphabetically.

With this algorithm of gathering the candidate list done, it was simple to pick a candidate out of the list. It would be moved to the top of the list and the whole list would be saved if the user gave the same prefix.

This project helped me a lot with understanding tries and trees as a data structure. I learned how to traverse these massive trees and follow certain paths based on the prefix. It was fun to complete this project as it is a way to apply a data structure in a useful way that could be implemented in a full project. If the user wanted to type something, giving them options of what to fill it in with would help with user satisfaction and efficency for the user.
